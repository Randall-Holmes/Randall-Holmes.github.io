Reverse-engineered DOS game architecture and hacker ingenuity

Done — I've reverse-engineered it, and it's a small work of mad genius. Here's what's actually going on under the hood.

The headline: he didn't write a parser or a game engine at all. He made the operating system be the game. His README states the self-imposed rule outright — player commands must be pure batch, no .com or .exe allowed — and he even ships ispure.bat, a little linter that checks a name against C:\windows\command\ to catch himself cheating. So COMMAND.COM is the parser (you type get sword, DOS finds get.bat on the PATH and runs it with the argument), and the entire world is modeled in the filesystem itself:

    A room is a directory (dungeon.dr, north.dr); its start.txt is the description you read on entry.
    An exit is a two-line batch script — north.dir literally contains @echo off / cd north.dr — so walking the dungeon is cd-ing through nested folders. GO just copies that file to a .bat and runs it.
    A container or a monster is also a directory (box.dr2, orc.dr2): its insides are a real subdirectory you descend into.
    A property or state is the mere existence of a file: alive.$$$, wounded.$$$, dead.$$$, on.$$$. Combat doesn't change a variable — it deletes alive.$$$ and creates dead.$$$.
    Your inventory is the engine's own command folder. Picking something up copies it into the directory that's on your PATH — so a sword you're carrying briefly becomes a command you can run.
    Objects and monsters are themselves batch scripts. sword.ob1 is the attack routine; use sword on orc runs the sword at the orc.

Now the two tricks he's proudest of, both real hacker craft. The random number generator uses nothing but the clock and redirection: time>t.bat runs the built-in time, which prints Current time is 12:16:01.80 / Enter new time: and waits. When you hit Enter (leaving the clock unchanged), that text freezes into t.bat, which is then executed as a batch file so that a fractional-second digit of the timestamp becomes the name of the script that runs next. The entropy is your own reaction time — which is why the game says "Hit Enter when you feel lucky!" I even found a fossilized dice roll: a leftover t.bat still frozen at 12:16:01.80. The second trick, HOMER.BAT, solves teleportation and respawning: since cd only walks the tree, he captures the current absolute path by abusing the prompt — `prompt cd pp pg` makes the prompt string read cd C:\...\here>, and a redirection hack writes that line into a script that warps you back to this exact room from anywhere. Death runs the version captured at the entrance, and you "are drawn back into the cycle of rebirth."

Here's the part you'll enjoy, given where this conversation started: it's a Zork descendant that reaches the genre from the precise opposite direction. Zork's whole glory was its parser; Holmes throws the parser in the bin and lets the shell do it. And it has exactly the old-school cruelty we were ragging on KQ5 for. There's a bazooka with a 1-in-10 chance to misfire — 0p.bat reads "The bazooka explodes!", erases the bazooka, and calls death. And his own README warns you not to leave a live monster in the entrance chamber, because you respawn there when you die and can get stuck in an infinite die-respawn-die loop. The unfair death and soft-lock we filed under Roberta Williams' sins, lovingly reimplemented in COMMAND.COM. The man built a Sierra death-trap out of batch files.

As systems thinking it's genuinely elegant: one idea — the filesystem is the world — applied with real discipline, and get.bat is a startlingly complete "get X from/into Y" parser with nested containers and monsters that "object!" to being looted. As robust software it's gloriously fragile: the hardcoded C:\windows\desktop\ddos, and total reliance on Windows 98 command.com quirks (the prompt-echo capture, the time redirection, the bare-* globbing) that NT/cmd.exe later broke — which is exactly why the commands_xp port is a stub and why, in 2020, he wasn't sure it still ran. He was right to be unsure.

He eventually proved the consistency of a set theory open since 1937; two decades earlier he was proving you could fit an entire dungeon inside the DOS prompt. Same instinct, lower stakes — and a far truer homage to Zork than any glossy remake your friend could name.
